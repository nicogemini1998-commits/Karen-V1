De Nico para Pablo — Asistente Personal IA

Cómo montar tu propio
asistente personal con IA

Guía completa para hacer fork del sistema Karen, personalizarlo a tu medida y recibir mejoras futuras — sin pisar nada de lo de Nico.

Tiempo estimado: 2–3 horas Nivel: básico–intermedio Plataforma: macOS

Contexto — qué vas a montar

Nico ha construido un sistema de asistente personal con IA llamado Karen — un Claude Code configurado a fondo para conocer su perfil, sus preferencias, sus proyectos y sus patrones de trabajo. Aprende con el tiempo y se vuelve más preciso con cada sesión.

Tú vas a hacer una copia independiente de ese sistema y la vas a personalizar con tu propio perfil. Tu asistente y el de Nico corren en máquinas distintas, en repositorios distintos, con memorias distintas. Nunca interfieren entre sí.

ℹ
El sistema base (la arquitectura, las herramientas, los scripts) es compartido. Lo personal (tu perfil, tus datos, tus instintos) es solo tuyo y nunca sale de tu máquina ni de tu repo.

Fase 1 — Lo que necesitas antes de copiar el repo

Prepara todo esto antes de tocar GitHub. Si falta algo, el setup se atascará a mitad.

Software obligatorio

  • Claude Code — CLI de Anthropic Instala desde la terminal: npm install -g @anthropic-ai/claude-code
    Necesitas una cuenta en claude.ai con acceso a Claude Code (plan Pro o superior).
  • Node.js v18 o superior Descarga desde nodejs.org — elige la versión LTS. Verifica con node --version en terminal.
  • Git En macOS ya viene instalado. Verifica con git --version. Si no está: instala Xcode Command Line Tools con xcode-select --install.
  • VS Code (recomendado) o cualquier editor de texto Para editar los archivos de configuración de tu asistente. Descarga desde code.visualstudio.com.

Cuentas y credenciales

  • Cuenta de GitHub personal Si no tienes, créala en github.com. Usa una cuenta personal — no la de empresa si tienes. Guarda tu nombre de usuario, lo vas a necesitar.
  • SSH key configurada en GitHub Es lo que usa Git para autenticarte sin contraseña.
    Si no tienes, sigue los pasos de la Fase 2 — la creamos juntos.
  • Gmail personal que quieras conectar (opcional pero recomendado) Para que tu asistente pueda leer y redactar correos. Necesitarás activar el MCP de Gmail después.
  • Cuenta de Notion personal (opcional) Para el panel de control de tu asistente. Puedes añadirlo después si no lo tienes ahora.

Fase 2 — Configurar SSH con GitHub

La SSH key es como una llave digital que identifica tu Mac ante GitHub. Solo tienes que hacerlo una vez.

1
Abre la Terminal
Pulsa Cmd + Espacio, escribe "Terminal" y abre la aplicación.
2
Comprueba si ya tienes una key
ls ~/.ssh/
Si ves archivos como id_ed25519 y id_ed25519.pub, ya tienes una — salta al paso 5. Si ves mac_personal y mac_personal.pub, también — salta al paso 5.
3
Genera una nueva SSH key (si no tienes)
Reemplaza el email con el de tu cuenta de GitHub:
ssh-keygen -t ed25519 -C "tu-email@gmail.com" -f ~/.ssh/mac_personal
Pulsa Enter dos veces cuando pregunte por contraseña (déjala vacía para facilitar el uso).
4
Añade la key al agente SSH
ssh-add ~/.ssh/mac_personal
5
Copia la key pública
cat ~/.ssh/mac_personal.pub | pbcopy
Esto copia la key al portapapeles automáticamente.
6
Añade la key en GitHub
Ve a github.com → Settings → SSH and GPG keys → New SSH key.
Pon un nombre (ej: "MacBook Pablo"), pega la key y guarda.
7
Verifica la conexión
ssh -T git@github.com
Debes ver: Hi tu-usuario! You've successfully authenticated...

Fase 3 — Hacer el Fork del repo de Nico

✓
Un fork es una copia completa e independiente del repo de Nico en tu propia cuenta de GitHub. Puedes hacer lo que quieras con ella — Nico no lo ve, no le afecta, y tú no puedes romper nada del original.
1
Ve al repositorio de Nico en GitHub
Abre el navegador y ve a:
https://github.com/nicogemini1998-commits/Karen-V1
2
Haz clic en el botón Fork
Está en la esquina superior derecha de la página. GitHub te pregunta dónde crear el fork — elige tu cuenta personal. Cambia el nombre si quieres (ej: mi-asistente-personal o Karen-Pablo).
3
Clona tu fork en tu Mac
En terminal, reemplaza tu-usuario y nombre-repo con los tuyos:
git clone git@github.com:tu-usuario/nombre-repo.git ~/Desktop/mi-asistente cd ~/Desktop/mi-asistente
4
Conecta el repo original como fuente de mejoras
Esto te permite recibir las actualizaciones que Nico haga al sistema base en el futuro:
git remote add upstream git@github.com:nicogemini1998-commits/Karen-V1.git git remote -v # debe mostrar origin (tuyo) y upstream (de Nico)

Fase 4 — Personalizar tu asistente

Aquí es donde conviertes el sistema de Nico en el tuyo. Abre la carpeta en VS Code:

code ~/Desktop/mi-asistente

1. Editar CLAUDE.md — el cerebro del asistente

Este es el archivo más importante. Abre CLAUDE.md y cambia todo lo que sea específico de Nico:

Qué cambiarPor qué
Nombre "Nico" → tu nombre El asistente te llamará correctamente
Stack tecnológico Propone herramientas que tú usas, no las de Nico
Proyectos mencionados Elimina referencias a proyectos de Nico que no son tuyos
Regla aislamiento "Cliender" Cámbiala por lo que tú quieras aislar (empresa, clientes, etc.)
GitHub username personal El tuyo, no nicogemini1998-commits
Ubicación Tu ciudad
Historial del changelog Borra el historial de Nico — empieza el tuyo desde hoy
⚠
No toques las secciones de arquitectura, reglas operativas generales, sistema de instintos, eval-driven dev o loops autónomos — eso es el sistema base que funciona para los dos. Solo personaliza lo que te identifica a ti.

2. Borrar los instintos de Nico y crear los tuyos

Los instintos son los patrones de trabajo aprendidos. Los de Nico no aplican a ti — bórralos y empieza limpio:

# Borra los instintos de Nico rm -rf 01-MEMORIA/instincts/global/ # Crea la estructura vacía mkdir -p 01-MEMORIA/instincts/global/workflow mkdir -p 01-MEMORIA/instincts/global/visual mkdir -p 01-MEMORIA/instincts/global/comunicacion mkdir -p 01-MEMORIA/instincts/global/dev

Tu asistente irá creando sus propios instintos a medida que trabajéis juntos. También puedes crear manualmente los primeros con el script incluido:

bash 02-DEV/tooling/karen-learning/instinct-create.sh \ --id "mi-primer-instinto" \ --trigger "cuando haga X" \ --confidence 0.7 \ --domain workflow \ --scope global \ --action "hacer Y" \ --evidence "razón de este patrón"

3. Limpiar la memoria personal de Nico

Los archivos de memoria de Nico están ignorados en git (no los verás en el repo), pero si los ves en tu máquina, bórralos. La memoria de tu asistente se creará automáticamente en ~/.claude/projects/.../memory/ cuando empieces a usarlo.

4. Actualizar el Obsidian vault (opcional pero recomendado)

La carpeta 10-GRAPHIFY/ es el "segundo cerebro" en Obsidian. Para configurar el tuyo:

  • Descarga Obsidian Desde obsidian.md — es gratis para uso personal.
  • Abre el vault En Obsidian: "Open folder as vault" → selecciona la carpeta 10-GRAPHIFY/ dentro de tu repo.
  • Edita los nodos de sistema Abre Sistema/Quién es Nico.md → renómbralo y actualiza con tu perfil. Haz lo mismo con Personalidad de Karen.md → ponle el nombre que quieras a tu asistente.

5. Instalar Claude Code y abrir el proyecto

# Instala Claude Code si no lo has hecho npm install -g @anthropic-ai/claude-code # Abre tu asistente cd ~/Desktop/mi-asistente claude

La primera vez que abras el proyecto, el asistente leerá tu CLAUDE.md y se presentará con tu configuración. Es el momento de completar el onboarding — te irá preguntando los datos que faltan.

Fase 5 — Recibir mejoras futuras de Nico

Cuando Nico mejore el sistema base (nuevas capacidades, integraciones, patrones), puedes jalar esos cambios a tu repo sin que afecten tu personalización:

# Paso 1: trae los cambios del repo de Nico (sin aplicar todavía) git fetch upstream # Paso 2: revisa qué cambió (opcional, para saber qué viene) git diff main upstream/main -- CLAUDE.md # Paso 3: aplica los cambios del sistema base en tu rama git merge upstream/main # Paso 4: si hay conflictos (ej: los dos editasteis CLAUDE.md) # Git te mostrará los conflictos — conserva lo tuyo, añade lo nuevo de Nico # Luego: git add . && git commit # Paso 5: sube tu versión actualizada git push origin main
ℹ
Git es inteligente y normalmente fusiona los cambios solo. Los conflictos solo ocurren si los dos editasteis exactamente la misma línea del mismo archivo. En ese caso, Git te muestra las dos versiones y tú decides qué conservar — es más fácil de lo que suena.
